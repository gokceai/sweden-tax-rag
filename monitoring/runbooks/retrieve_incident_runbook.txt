Sweden Tax RAG Incident Runbook: Retrieve 500/503

1) Confirm API is up
- GET /health/live
- Expected: status=ok

2) Check readiness and deep health
- GET /health/ready
- GET /health/deep
- If 503, inspect checks.vector_db / checks.dynamo_db fields.

3) Validate dependencies
- Chroma: reachable on configured host/port
- Dynamo: reachable on configured endpoint and table exists
- Check env vars: CHROMA_HOST, CHROMA_PORT, DYNAMO_ENDPOINT, MASTER_ENCRYPTION_KEY

4) Inspect metrics
- /metrics should include:
  - http_requests_total
  - rag_retrieve_requests_total
  - rag_retrieve_duration_seconds
- Check 5xx spike and latency p95 in Grafana.

5) Common fixes
- Chroma connection errors: restart chroma container, verify mount path and collection config
- Dynamo connection errors: restart dynamodb-local, verify endpoint and credentials
- Encryption errors: ensure MASTER_ENCRYPTION_KEY is valid Fernet key

6) Validate recovery
- Re-run /health/ready and /health/deep
- Ask a known query via /api/v1/retrieve
- Confirm 5xx drops and p95 stabilizes
